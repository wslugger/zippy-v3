# **Zippy v3 Project Index**

This index serves as the master map for the Zippy v3 architectural foundation. It is organized into active behavioral rules for the AI agent and reference blueprints for the system design.

## **🤖 AI Agent Rules (.agent/rules/)**

*These files dictate the AI's persona, coding standards, and mandatory business constraints.*

* [**claude.md**](https://www.google.com/search?q=./.agent/rules/claude.md): The "Project Constitution." Contains the Zippy persona, the 4-module SA Flow, iterative state invalidation rules, and critical anti-patterns.  
* [**UI\_STANDARDS.md**](https://www.google.com/search?q=./.agent/rules/UI_STANDARDS.md): Frontend mandate. Enforces Tailwind CSS v4, shadcn/ui (owned code), high-density data layouts, and semantic state colors.

## **📚 Reference Blueprints (docs/)**

*These files provide the technical specifications for the database, API, and system logic.*

* [**ARCHITECTURE\_v3.md**](https://www.google.com/search?q=./docs/ARCHITECTURE_v3.md): The high-level system wiring. Details the Federated Monolith architecture and the Project Ledger (MongoDB) embedding strategy.  
* [**CATALOG\_DATA\_MODEL.md**](https://www.google.com/search?q=./docs/CATALOG_DATA_MODEL.md): Polymorphic schema definitions for Equipment, Services, and Packages. Includes the **Lean Actionable Specs** mandate and HLD text blocks.  
* [**BOM\_LOGIC\_ENGINE.md**](https://www.google.com/search?q=./docs/BOM_LOGIC_ENGINE.md): Specifications for the stateless Policy Resolution API, JSON-logic rule structures, and the explainability (reasoning) mandate.  
* [**GLOBAL\_TAXONOMY.md**](https://www.google.com/search?q=./docs/GLOBAL_TAXONOMY.md): Blueprint for the singleton system configuration document that manages shared vocabularies (vendors, PoE standards, etc.).
